2) Savoir choisir et utiliser les balises les plus courantes

Balises de premier niveau

Tout contenu doit être inclus dans une balise. Il existe des balises de premier niveau, elles sont inclues directement dans la balise body. Chaque balise doit être choisie avec précaution. Chacune a un rôle précis déterminé par le W3C (on en reparlera plus tard).

Voici les deux premières balises que vous devez connaitre (rôle, et rendu visuel):

  • des titres de tailles différentes permettant de scructurer votre page : h1, h2, h3 ...
  • des paragraphes pour écrire du texte : p
    • <body>
    <h1>L'IUT de Bordeaux</h1>
    <h2>Le département GMP</h2>
    <h3>Présentation</h3>
    <p>Quelques détails sur le département GMP</p>
    <h3>Modalités d'amission</h3>
    <p>Quelques détails sur l'admission en GMP</p>
    <h2>Le département INFO</h2>
    <h3>Présentation</h3>
    <p>Quelques détails sur le département INFO</p>
    <h3>Modalités d'amission</h3>
    <p>Quelques détails sur l'admission en INFO</p>
</body>

Après avoir copié/collé ce code dans un fichier, regardez le vidéo suivante et observez que :
  • A chaque balise est associé un style par défaut (taille, couleur, fonte, poistionnement dans la page, ...)
  • Qu'il est possible, via le debugger, d'avoir accès à toutes les informations d'affichage de chaque balise
  • Qu'il est possible de modifier le contenu de la page depuis le debugger (mais ne modifie pas votre fichier source)

Autres balises

Afin de pouvoir générer un contenu visuel un peu plus riche que le permet la balise p nous allons voir 4 nouvelles balises permettant de : générer des listes, intégrer des images, intégrer des liens hypertextes, créer des tableaux.

Les listes

ul et li permettent de structurer des listes d’éléments. ul signifie "unsorted list", liste non-ordonnée.

<ul>
    <li>un element</li>
    <li>un autre</li>
</ul>
  • un élément
  • un autre

Il existe plusieurs manières de créer des listes. Allez lire la documentation officielle de ol et dl/dt

Un lien hypertexte : la balise a

créer des liens en HTML,vous aurez à minima besoin d'une cible et d'un texte :

  • cible : L'URL de la destination du lien (par exemple, "https://www.wikipedia.com")
  • texte : Le texte qui décrit la destination offerte par le lien.
Le texte sera placé entre les balises et la cible sera la valeur de l'attribut href . 

<p>
Un lien vers <a href="https://www.wikipedia.com">une encycopédie en ligne</a> !
</p>

Un lien vers une encycopédie en ligne !

Il est également possible de créer un lien qui pointe vers un endroit spécifique d'un document. Cet endroit est appelé une ancre. Ajoutez un attribut id à l'élément cible. Par exemple, la balise 
<h5>Balises à intégrer dans un paragraphe</h5>
devient 
<h5 id="ancre">Balises à intégrer dans un paragraphe</h5>
Dans l'URL du lien, placée dans l'attribut `href`, ajoutez l'identifiant précédé d'un dièse (#) : 
<p>Cliquer <a href="#ancre">ici</a> pour se rendre sur le paragraphe parlant des balises </p>

Cliquer ici pour se rendre sur le paragraphe parlant des balises.

La balise image

L'élément HTML img permet d'intégrer une image dans un document. L'attribut alt n'est pas obligatoire mais recommandé pour des raisons d'accessibilité, en cas de problème de chargement de l'image, pour l'indexation de votre page par des robots. alt est une description textuelle de l'image. contient une description textuelle de l'image. L'attribut src est obligatoire et contient le chemin de l'image.

La baslie img est ce qu'on appelle une "balise vide". Elle n'a pas de balise fermante (car pas de contenu).

src peut être un chemin vers une image sur internet ou un chemin vers vos propres répertoires.

Attention : en S1 nous travaillons sans serveur web. Le chemin vers une image doit être un chemin relatif à la page html. Le chemin doit donc avoir la forme suivante src="img/seance1/portrait.jpg" . Il est donc interdit d'écrire un chemin absolu de la forme src="/Users/nicholasjournet/cours/coursVersionnes/webs1/img/seance1/portrait.jpg"

Attention : il est possible de redimensionner à la volée l'image avec l'attribut width et l'attribut height de la balise img. En S1, préférez TOUJOURS retailler l'image au préalable (avec un logiciel de traitement d'image) 

<img src="https://uploads7.wikiart.org/images/gustave-courbet.jpg!Portrait.jpg" alt="Le désespéré, Gustave Courbet" />
<img src="img/seance1/portrait.jpg" alt="Le désespéré, Gustave Courbet" />

Les tableaux

Une table HTML est définie avec la balise table. Si elle contient des entêtes (des labels)talors il faut utilser les balises thead et tBody pour différencier les entêtes du corps du tableau. Chaque ligne du tableau est définie avec la balise tr. Un en-tête de table est défini (dans thead) avec la balise th. Par défaut, les en-têtes de tableau sont en gras et centrés. Une cellule est définie avec la balise td. Les éléments td sont les conteneurs de données de la table. Ils peuvent contenir toutes sortes d’éléments HTML : texte, images, listes, autres tableaux, etc. 

<table>
    <thead>
        <tr>
            <th>prenom</th>
            <th>nom</th>
            <th>Age</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Sarah</td>
            <td>Bernard</td> 
            <td>50</td>
        </tr>
        <tr>
            <td>Jane</td>
            <td>Doe</td> 
            <td>?</td>
        </tr>
    </tbody>
</table>

Ajouter de la sémantique à votre page

Afin de donner de la sémantique à votre code et optimiser votre référencement, vous pouvez utiliser les deux balises génériques div et span. Nous verrons lors de la prochaine séance, que cela permet également de référencer du code qui sera stylisé à l'aide du CSS.

Chaque balise HTML possède deux attributs :

  • id : identifiant de la balise qui doit être UNIQUE sur l'ensemble de votre code
  • class : valeur qui peut être attribuée à autant de balises que vous le souhaitez

Par exemple le code 
<!-- code de mon menu (unique)-->
    <ul>
        <li> menu 1 </li>
        <li> menu 2 </li>
    </ul>

<p>
    <!-- code de mon premier article -->
    Ceci est article parlant du département informatique de l'IUT de Bordeaux.
</p>                

<p>
    <!-- code de mon second article -->
    Ceci est article parlant du département HSE de l'IUT de Bordeaux.
</p>
Peut être ré-écrit de la manière suivante sans que cela change ce qui est généré : 

<div id="menu">
        <!-- code de mon menu (unique)-->
        <ul>
            <li> menu 1 </li>
            <li> menu 2 </li>
        </ul>         
</div>
<div id="mesArticles">
    <div class="unArticle">
        <p>
         <!-- code de mon premier article -->
         Ceci est article parlant du <span class="nomDepartement">département informatique</span> de l'IUT de Bordeaux.
        </p>                
    </div>
    <div class="unArticle">
        <p>
         <!-- code de mon second article -->
         Ceci est article parlant du <span class="nomDepartement">département HSE</span> de l'IUT de Bordeaux.
        </p>
    </div>
</div>

La norme HTML5 introduit de nouveaux éléments inédits dotés de sens et apportant une alternative aux blocs génériques div et span pour lesquels les développeurs ont tendance à choisir des noms d'identifiants ou de classes peu explicites et dans leur langue.

Ce balisage explicite aide les robots (comme ceux de Google, de Bing , de Qwant, ...) à comprendre les rôles et l'importance relative des différentes parties de votre page, et donc, améliore le référencement naturel de vos pages (SEO : « search engine optimisation »).

Ces balises sont :

  • header (entête du site)
  • footer (pied de page du site)
  • main (partie principale où se situe le contenu)
  • section (regroupe des contenus thématiques)
  • article (composition autonome dans un document)
  • nav (généralement le menu)
  • aside (indique que son contenu n'a qu'un rapport indirect avec le contenu principal du document.)
  • figure (utilisé pour y intégrer un contenu comme une image ou une vidéo)
L'image ci-dessous illustre une utilisation possible de ces balises.

Le code précédent peut être ré-écrit de la manière suivante : 
<header>
    <nav>
            <!-- code de mon menu (unique)-->
            <ul>
                <li> menu 1 </li>
                <li> menu 2 </li>
            </ul>                
    </nav>
</header>
<section>
    <article>
        <p>
            <!-- code de mon premier article -->
            Ceci est article parlant du <span class="nomDepartement">département informatique</span> de l'IUT de Bordeaux.
        </p>                
    </article>
    <article>
        <p>
            <!-- code de mon second article -->
            Ceci est article parlant du <span class="nomDepartement">département HSE</span> de l'IUT de Bordeaux.
        </p>
    </article>
</section>

3) Savoir tester la qualité de son code

Installez un linter

Ecrire un code de qualité commence par l'installation d'un linter. Un linter est un module que vous pouvez paramétrer afin qu'il vous indique, lorsque vous codez, s'il détecte des erreurs, ou du code de mauvaise qualité (code non utilisé, non respect des standards, ...). Il en existe plein. Vous pouvez utiliser le module VSCode HTMLHint (cliquez sur le picto "extension" dans le menu de gauche).

Valider son code

Le consortium W3C qui émet les normes du web (donc HTML) fournit un outil en ligne dans lequel vous pouvez copier/coller votre code.
