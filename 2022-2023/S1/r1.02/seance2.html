Une déclaration CSS peut être mise dans la partie head de votre code, ou dans un fichier indépendant (extension .css qu'il faut inclure ). 
<!doctype html>
<html lang="fr">
<head>
    <meta charset="utf-8">
    <title>Titre de la page</title>
    <!-- soultion 1 un lien vers une ressource externe de type css -->
    <link href="css/style.css" rel="stylesheet">
    <style>
    /*solution 2 : définir du style CSS dans le html*/
    </style>
</head>
<body>
    ...
    <!-- Le reste du contenu -->
    ...
</body>
</html>

Une déclaration css comporte un sélecteur suivi d’une liste de couples « propriété valeur »:

sélecteur {
    propriété1 : une valeur ;
    propriété2 : une valeur ;
    ...
    propriétén : une valeur ;
    }

En CSS, un sélecteur est donc utilisé pour cibler une(des) partie(s) spécifique(s) d'une page web à mettre en forme. Afin de pouvoir être précis, CSS est très riche en sélecteurs, des premiers sélecteurs du CSS1 aux pseudos éléments de CSS3, en attendant les dernières nouveautés promises par CSS4.

Sélecteurs de balises

CSS n'est pas un langage de programmation, il permet simplement de mettre en forme le contenu de la page HTML. Si votre sélecteur se résume au nom d'une balide html, alors toutes les balises de ce type seront sélectionnées et le style décrit entre accolades sera appliqué.

<style>    
    /*sélectionner toutes les balises p dans le code HTML et mettre le texte en rouge et taille 12px;*/
p {
    color : red;
    font-size : 10px;
}

/*sélectionner toutes les balises ul et afficher un carré pour chaque item au lieu d'un cercle*/
ul {
    list-style: square;
}
</style>
<body>
    <h1>Titre</h1>
    <p>paragraphe 1</p>
    <ul>
        <li>item 1</li>
        <li>item 2</li>
    </ul>
    <p>paragraphe 2</p>
</body>

Il est important de savoir manipuler le debugger de votre navigateur. Comme sur la vidéo, testez le code précédent et modifiez à la volée les valeurs des paramètres.

Sélecteurs d'id et de class

Le sélecteur . permet de sélectionner tous les élements d'une classe donnée. Le sélecteur # sélection l'élément d'un identifiant.

<style>    
/*sélectionner toutes les balises de classe "unArticle" (donc 2 ici)*/
.unArticle {
    color : red;
    font-size : 10px;
}

/*sélectionner la balise d'id "menu" (donc une seule)*/
#menu {
    list-style: square;
}
</style>
<body>
    <ul id="menu">
        <li>item 1</li>
        <li>item 2</li>
    </ul>
    <h1>Titre</h1>
    <p class="unArticle">article 1</p>
    <p >Publicité !</p>
    <p class="unArticle">article 2</p>
    <p >Un autre Publicité !</p>
    <ul id="footer">
        <li>item 1</li>
        <li>item 2</li>
    </ul>
    
    <p>paragraphe 2</p>
    </body>

Il est possible de spécifier sur quel type de balise le sélecteur de classe est appliqué

<style>    
    /*sélectionner toutes les balises p de classe "unArticle" (donc 2 ici)*/
    /*Donc h1, même s'il est de classe article n'est pas sélectionné*/
    p.unArticle {
        color : red;
        font-size : 10px;
    }
    </style>
    <body>
        <h1 class="unArticle">Titre</h1>
        <p class="unArticle">article 1</p>
        <p >Publicité !</p>
        <p class="unArticle">article 2</p>
        <p >Un autre Publicité !</p>        
</body>

Des sélecteurs plus évolués

Combiner des sélecteurs

Pour appliquer le même style à plusieurs éléments, il est possible de combiner plusieurs sélecteurs en les séparant par une virgule , 

<style>    
/*sélectionner toutes les balises et toutes les balises h2 et tous le paragraphes de classe "monArticle*/
h1, h2, p.monArticle {
    color : red;
}
</style>

Sélecteurs d'adjacences X~Y et X+Y

Dans l'exemple ci-dessous :

  • div et span 5 sont frères directs car ils se suivent et sont de même niveau
  • span 1, le paragraphe de classe paragraphe et les span 3 et span 4 sont frères. Ils ont le même père qui est la balise div
  • p est le frère direct de span 1, span 3 est le frère direct de p, span 4 est le frère direct de span 3.
  • span 2 n'a pas de frère.
<div>
    <span> span 1 </span>
    <p class="paragraphe">
            <span> span2 </span>
    </p>
    <span> span 3 </span>
    <span> span 4 </span>
</div>
<span> span 5 </span>

Le sélecteur X+Y sélectionne toutes les balises Y frères direct d'une balise X (ils doivent avoir le même père et se suivrent dans le code).

Le sélecteur X~Y sélectionne toutes les balises Y frères indirect d'une balise X (ils doivent avoir le même père mais ne sont pas obligées de se suivrent dans le code). 

/*sélectionne toutes les span qui ont un (grand) frère direct de type p */
p + span {color:red;}/*sélectionne span 3*/

/*sélectionne toutes les span qui ont un (grand) frère direct de type span */
span + span {color : red;}/*sélectionne span 4*/

/*sélectionne toutes les span qui ont un (grand) frère (mais pas obligatoirement direct) de type p */
p ~ span {color:red;}/*sélectionne span 3*/

/*sélectionne toutes les span qui ont un (grand) frère (mais pas obligatoirement direct) de type span */
span ~ span {color : red;}/*sélectionne span 3 et span 4*/

Sélecteur d’attribut [ ]

Chaque élément HTML peut avoir un ou plusieurs attributs. Ces attributs sont des caractéristiques supplémentaires (possédant souvent une plage de valeurs possibles) qui permettent de configurer l’élément ou de préciser son comportement. Par exemple : 

<input type="text" required>

[name] est le sélecteur d’attribut qui permet de cibler toutes les balises de la page qui utilisent cet attribut. 

/*sélectionner tous les input de type text*/
input[type="text"] {
    /*attributs/valeurs*/
    }
/*sélectionner toutes les ancres possédant un titre et toutes les ancres  dont href contient le terme "IUT"*/
a[href*="IUT"], a[title] { 
    background-color: red; 
}

Il est possible de cibler suivant tout ou partie de la valeur de l’attribut. Vous pourrez utiliser les symboles =, ~=, |=, ^=, ¤=, *= qui ont chacun une spécificité.

les pseudo-classes et pseudo-éléments :

Une pseudo-classe est un mot-clé qui peut être ajouté à un sélecteur afin d'indiquer l'état spécifique dans lequel l'élément doit être pour être ciblé par la déclaration. La pseudo-classe :hover, par exemple, permettra d'appliquer une mise en forme spécifique lorsque l'utilisateur survole l'élément ciblé par le sélecteur (changer la couleur d'un bouton par exemple).

Il existe une trentaine de pseudo-classes : active, checked, first-child, :hover, ...

Les pseudo-éléments CSS vont permettre de modifier l’apparence d’une partie seulement d’un élément HTML, ou encore d’ajouter du contenu au début ou à la fin d'un certain élément HTML**. On peut citer notamment les pseudo-éléments ::first-letter, ::first-line, ::before, ::after. Cascade, héritage et poids des sélecteurs

Cascade

Le C de CSS signifie cascade. Comme défini par mozilla , la façon dont la cascade se comporte est la clé de la compréhension du CSS. La cascade des styles signifie que l'ordre d'apparition des règles dans le CSS a une importance ; quand deux règles applicables ont la même spécificité, c'est la dernière déclarée qui sera utilisée pour la mise en forme. Dans l'exemple ci-dessous tous les h1 seront bleus. 

h1 { 
    color: red; 
}
h1 { 
    color: blue; 
}

Héritage

Comme défini par mozilla : L'héritage transmet des valeurs de propriétés depuis des éléments parents vers leurs enfants. Certaines , propriétés ne s'héritent pas.

Par exemple, si vous fixez une color et une font-family pour un élément, tout élément contenu dans le premier sera mis en forme avec la même couleur et la même police, à moins qu'on lui ait appliqué directement des règles. 

<style>
body {
    color: blue;
}
span {
    color: black;
}
</style>
<body>
    <p>
        Lorem ipsum dolor, sit amet consectetur adipisicing elit. Consequatur consectetur beatae sunt inventore expedita quod assumenda excepturi laudantium porro voluptas, commodi odio, odit dolore corporis, doloremque officia! Ratione, minus nesciunt!
    </p>
    <p>
        Lorem <span>ipsum</span> dolor, sit amet consectetur adipisicing elit. Consequatur consectetur beatae sunt inventore expedita quod assumenda excepturi laudantium porro voluptas, commodi odio, odit dolore corporis, doloremque officia! Ratione, minus nesciunt!
    </p>
</body>

donne le résultat suivant :

Comme on peut le voir dans le debugger, la balise p a hérité de la propriété color de body, car p est fils de body et la propriété color s'hérite

La propriété color de span a été explicitement redéfinie. Cette règle surcharge donc celle héritée via body

Poids des sélecteurs

Les spécifications CSS proposent une classification des sélecteurs selon un barème de poids sous forme de notation à quatre chiffres :

  • Poids a : règle CSS déclarée à l’aide de l’attribut HTML style="" au sein de l'élément
  • Poids b : sélecteur d’identifiant
  • Poids c : sélecteur de classe, d’attribut ([]) ou de pseudo-classes ( :hover, :focus, :firstchild)
  • Poids d : sélecteurs d'élément (p, div, a, ...) ou de pseudo-élément ( :first-letter, :before, :after, ...)
  • poids nul : parenté ( > ) ou adjacence (+)
Par exemple : 
#content  nav li a.on:hover {
color: red;
}

1 ID (#content), 2 classe+pseudo-classe (.on :hover), 3 éléments (nav li a) donne le poids : 123

Il surpassera ce sélecteur: 

nav.verte li.gras a.on:hover {
    color: green;
}

0 ID, 4 classe+pseudo-classe (.on :hover .verte .gras), 3 éléments (nav li a) donne le poids : 43

mais pas celui-ci: 

#content #nav a {
      color: red;
    }

2 ID, 0 classe+pseudo-classe, 1 éléments (a) donne le poids : 201

La déclaration !important**

L'ajout d'un !important, entre la valeur et le ; permet de surpasser les règles de cascade et de spécificité (pour autant qu'il n'y ait pas d'autres valeurs comportant un !important auquel cas, on retrouve les règles de cascade et spécificités appliquées): 


#content nav a {
 color: red !important;
}
